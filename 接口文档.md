# Simple Mall接口文档

**统一规范（后端）：包名小写，类名首字母大写，变量方法名采用驼峰命名，实体类统一使用lombok注解**

**实体类id用实体类名称(无需考虑复数)+Id命名，其他属性用实体类首字母小写+属性命名。数据库命名同理**

例如：

User类： userId——用户id

​				 uId——用户账号

​				 uName——用户姓名

​				 uEmail——用户邮箱

## 1. 商品列表

### 1.1 商品列表查询

#### 1.1.1 基本信息

> 请求路径：/products
>
> 请求方式：GET
>
> 接口描述：用于分页查询商品列表数据



#### 1.1.2 请求参数

无



#### 1.1.3 响应数据

参数格式：JSON

参数说明：

| 参数名        | 类型       | 是否必须 | 备注                           |
| ------------- | ---------- | -------- | ------------------------------ |
| code          | number     | 必须     | 响应码，1 代表成功，0 代表失败 |
| msg           | string     | 非必须   | 提示信息                       |
| data          | object[ ]  | 必须     | 查询                           |
| \|- total     | number     | 必须     | 商品总数                       |
| \|- row       | object[ ]  | 非必须   | 商品数据                       |
| \|- productId | number     | 非必须   | 商品编号                       |
| \|- pImg      | string     | 非必须   | 商品图片                       |
| \|- pName     | string     | 非必须   | 商品名称                       |
| \|- pPrice    | BigDecimal | 非必须   | 商品价格                       |
| \|- pNumber   | number     | 非必须   | 商品库存                       |

响应数据样例：

```json
{
  "code": 1,
  "msg": "success",
  "data": [
      total:2
    {
      "productId": 1,
      "pImg":"owerroiqw341.png",
```

## 5. 其他接口

### 5.1 登录

#### 5.1.1 基本信息

> 请求路径：/login
>
> 请求方式：POST
>
> 接口描述：登录系统完毕后，系统下发JWT令牌



#### 5.1.2 请求参数

参数格式：JOSN

| 名称      | 类型   | 是否必须 | 备注     |
| --------- | ------ | -------- | -------- |
| uId       | string | 必须     | 用户账号 |
| uPassword | string | 必须     | 密码     |

请求数据样例：

```json
{
	"uId": "zhangsan",
    "uPassword": "123456"
}
```



#### 5.1.3 响应数据

参数格式：JSON

参数说明：

| 名称 | 类型   | 是否必须 | 默认值 | 备注                     | 其他信息 |
| ---- | ------ | -------- | ------ | ------------------------ | -------- |
| code | number | 必须     |        | 响应码, 1 成功 ; 0  失败 |          |
| msg  | string | 非必须   |        | 提示信息                 |          |
| data | string | 必须     |        | 返回的数据 , jwt令牌     |          |



响应数据样例：

```json
{
  "code": 1,
  "msg": "success",
  "data": "eyJhbGciOiJIUzI1NiJ9.eyJuYW1lIjoi6YeR5bq4IiwiaWQiOjEsInVzZXJuYW1lIjoiamlueW9uZyIsImV4cCI6MTY2MjIwNzA0OH0.KkUc_CXJZJ8Dd063eImx4H9Ojfrr6XMJ-yVzaWCVZCo"
}
```





#### 5.1.4 备注说明

> 用户登录成功后，系统会自动下发JWT令牌，然后在后续的每次请求中，都需要在请求头header中携带到服务端，请求头的名称为 token ，值为 登录时下发的JWT令牌。
>
> 如果检测到用户未登录，则会返回如下固定错误信息：
>
> ```json
> {
> 	"code": 0,
> 	"msg": "NOT_LOGIN",
> 	"data": null
> }
> ```
